# Glossary

Collect the core Validibot terms here so new users have a quick reference.

## Workflow
Describe how a workflow represents an ordered set of validation steps owned by an organization and scoped to a project.

## Project
Explain that projects group workflows, submissions, and runs inside an organization. Mention default projects and how colors/badges work.

## Submission
Define a submission as the content uploaded or posted via API before validation begins.

## Validation Run
Clarify that each submission triggers a run, which records step-by-step progress and findings.

## Validation Step
Note that steps link to validators or actions and execute in order.

## Roles
Summarize OWNER, ADMIN, AUTHOR, EXECUTOR responsibilities, with a reminder to see the collaboration guide for full details.
