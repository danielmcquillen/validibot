{% trans "What is the evidence hash?" %}

{% blocktrans trimmed %} The evidence hash is a SHA-256 fingerprint computed when a validation run completes. It covers the key fields of the run record, creating a tamper-evident seal. If any covered field were modified after completion, the hash would no longer match. {% endblocktrans %}

{% trans "Fields included" %}
  • run_id — {% trans "Unique identifier of the run" %}
  • content_hash — {% trans "SHA-256 of the submitted content" %}
  • user_id — {% trans "ID of the user who submitted the run" %}
  • submission_name — {% trans "Name of the submission (if provided)" %}
  • status — {% trans "Final status (e.g. Succeeded, Failed)" %}
  • started_at — {% trans "When the run started (ISO 8601 UTC)" %}
  • ended_at — {% trans "When the run completed (ISO 8601 UTC)" %}
  • duration_ms — {% trans "Run duration in milliseconds" %}
  • error_count — {% trans "Number of error-severity findings" %}
  • warning_count — {% trans "Number of warning-severity findings" %}
  • info_count — {% trans "Number of info-severity findings" %}
  • total_findings — {% trans "Total finding count" %}
{% trans "How it works" %}

{% blocktrans trimmed %} These fields are serialized as canonical JSON (sorted keys, no whitespace), then hashed with SHA-256. The resulting 64-character hex digest is stored on the run record and included in API responses. External systems can recompute the hash from the same fields to verify that the record has not been altered. {% endblocktrans %}
