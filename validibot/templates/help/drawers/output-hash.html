{% trans "What is the output hash?" %}

{% blocktrans trimmed %} The output hash is a SHA-256 fingerprint computed when a validation run completes. It covers the stable validation context and the final run outcome, creating a tamper-evident seal. If any covered field were modified after completion, the hash would no longer match. {% endblocktrans %}

{% trans "Fields included" %}
  • validation context — {% trans "The workflow or workflow-hash context used to interpret the run" %}
  • run_id — {% trans "Unique identifier of the run" %}
  • status — {% trans "Final status (e.g. Succeeded, Failed)" %}
  • started_at — {% trans "When the run started (ISO 8601 UTC)" %}
  • ended_at — {% trans "When the run completed (ISO 8601 UTC)" %}
  • duration_ms — {% trans "Run duration in milliseconds" %}
  • error_count — {% trans "Number of error-severity findings" %}
  • warning_count — {% trans "Number of warning-severity findings" %}
  • info_count — {% trans "Number of info-severity findings" %}
  • total_findings — {% trans "Total finding count" %}
{% trans "How it works" %}

{% blocktrans trimmed %} Validibot serializes the covered fields into a deterministic JSON document, hashes that document with SHA-256, and stores the resulting 64-character digest on the run record. Installations with signed credentials enabled use the stricter workflow-centered Layer 3 contract from Validibot Pro; community installs fall back to the built-in host contract. In both cases, the digest helps reveal whether the run record was changed after completion. {% endblocktrans %}
