{% trans "What the SHACL validator does" %}

{% blocktrans trimmed %} The SHACL validator checks an RDF submission against one or more SHACL shape files. It is the right validator for Turtle, JSON-LD, RDF/XML, N-Triples, and N-Quads when the acceptance rules are expressed as RDF graph constraints. {% endblocktrans %}

{% trans "The graphs involved" %}
  • {% trans "Submitted RDF data graph" %} - {% trans "the submitter's parsed model. Most SPARQL ASK assertions should query this graph." %}
  • {% trans "Shapes graph" %} - {% trans "the uploaded SHACL shapes that define the validation rules." %}
  • {% trans "Supplementary ontology graph" %} - {% trans "optional context for class and property inference." %}
  • {% trans "SHACL results graph" %} - {% trans "the validation report produced by SHACL. It contains result nodes with focus nodes, source shapes, result paths, severities, and messages." %}
{% trans "SHACL result handling" %}

{% blocktrans trimmed %} This setting controls how normal SHACL validation results affect the step outcome. It does not soften infrastructure failures: RDF parse errors, invalid shapes, timeouts, and engine errors always fail the step immediately. {% endblocktrans %}

  • {% trans "Fail immediately on violations" %} - {% trans "stop after SHACL validation if the report contains any violation-level result. Step assertions do not run." %}
  • {% trans "Fail after assertions" %} - {% trans "run SHACL first, then run the step assertions, then fail the step if SHACL reported violations or any assertion failed. This is the recommended default." %}
  • {% trans "Report only" %} - {% trans "do not convert SHACL validation results into blocking findings. Counts and the native report graph are still available, so SPARQL ASK, Basic, or CEL assertions can decide exactly what should fail." %}
{% trans "SPARQL ASK target graphs" %}

{% blocktrans trimmed %} SHACL assertions use SPARQL ASK because it is the standard way to ask true/false questions about RDF graphs. {% endblocktrans %}

  • {% trans "Submitted RDF data graph" %} - {% trans "ask whether the submitted model contains or omits required triples." %}
  • {% trans "SHACL results graph" %} - {% trans "ask about the validation report itself, such as whether a specific source shape produced a violation." %}
  • {% trans "Data + results graph" %} - {% trans "join report findings back to submitted resources when an assertion needs both contexts." %}
{% trans "Example results-graph assertion" %}

{% blocktrans trimmed %} This ASK passes only when no SHACL violation came from a particular shape. It queries the validation report, not the original submission. {% endblocktrans %} 

PREFIX sh: <http://www.w3.org/ns/shacl#>
PREFIX ex: <http://example.com/>

ASK {
  FILTER NOT EXISTS {
    ?result a sh:ValidationResult ;
            sh:resultSeverity sh:Violation ;
            sh:sourceShape ex:DamperShape .
  }
}
