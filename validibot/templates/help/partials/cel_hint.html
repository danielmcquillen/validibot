{% trans 'CEL Hints' %}

When creating a CEL expression you can either use the name of the target signals or the data path path directly.

Some validators only support one style or the other.

Generic examples of CEL expressions:

  • price > 0
  • thermostat[3].setpoint < 78

{% trans 'XML tip:' %} {% trans 'Numeric values need casting:' %} double(price) > 0

{% trans 'XML attributes:' %} {% trans 'Use the @ prefix with bracket notation:' %} m["@Conductivity"]
