{% load i18n core_tags %} {% comment %} Confirmation interstitial shown between the invite form and the actual send (``InviteConfirmView`` -> here -> ``InviteCreateView``). It spells out *who* is being invited and *which* permissions they'll receive so an admin can catch a misclick before org access is granted. Two flavours, decided by ``invitee_user``: * set -> a type-ahead pick from people this org already knows; show their identity card so the admin confirms the right person. * unset -> a raw-typed address. We deliberately do NOT reveal whether an account exists for it (that would be an account-enumeration oracle), so the copy stays neutral. ``save()`` binds an existing account server-side, without surfacing the result here. Expected context: organization, invitee_user (User|None), invitee_email, selected_roles (list of {code,label}), search_value, role_codes. {% endcomment %}
