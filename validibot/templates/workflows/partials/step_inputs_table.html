{% load i18n core_tags %} {% comment %} Step Inputs Table Shows all step inputs for a workflow step. Each row shows the input name, source path, default value, required indicator, the "Copy to Signal" promotion control, and an edit button. Per ADR-2026-05-22, step inputs live in the i.* CEL namespace and are populated by the validator's parser hook (e.g., EnergyPlus IDF parser) or by resolved StepInputBindings. Authors can promote any step input to a workflow signal (s.) so downstream steps can reference it — this mirrors the existing output-promotion mechanism. The outer div listens for the ``signals-changed`` HTMx event (fired by the promote/demote view) and re-fetches the table partial so the UI updates without a full page refresh. This was added by the July 2026 P2 review: previously input promotion succeeded server-side but the table didn't show the new s. badge or the remove button until a full page reload. The output table already had this wiring; the input table didn't. An alert bar warns when required inputs have no source path mapped. Context: unified_signals.input_signals - list of step input dicts step - the WorkflowStep workflow - the parent Workflow can_manage_workflow - whether the current user can edit {% endcomment %}
