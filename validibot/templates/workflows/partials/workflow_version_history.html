{% load i18n %} {% comment %} Workflow version-history panel. Renders the (org, slug) family of workflow versions as a compact table. This panel is the only authoring UI navigation surface for opening an older workflow version. It shows enough metadata (run count, policy, locked/archived qualifiers, timestamps) for authors to understand what each version is for before clicking through. Context expected (built by ``build_workflow_version_context``): - ``workflow_versions``: list of dicts, newest-first, each with ``version``, ``url``, ``is_current``, ``is_latest``, ``is_active``, ``is_archived``, ``is_tombstoned``, ``is_locked``, ``run_count``, ``history_policy``, ``created``, ``modified``. - ``has_workflow_versions``: True when there are 2 or more versions. When there's only one version in the family the panel hides itself entirely. A single version is its own history — there's nothing to switch to and nothing useful to compare against. {% endcomment %} {% if has_workflow_versions %}
