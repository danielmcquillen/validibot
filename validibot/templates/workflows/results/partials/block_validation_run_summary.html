{% load i18n core_tags %} {% comment %} Run Summary card. Displays run metadata: status, submission name, finding counts, source, timestamps, and duration. Submission content and metadata are shown in the User Inputs card instead. Context variables (passed via ``{% include ... with %}``): run — the ValidationRun object. bare — when truthy, render only the inner content (no card / header / body chrome). The caller supplies the surrounding chrome — used by the stacked accordion layout where the accordion is the chrome. Defaults to falsy, preserving the standalone card for the classic layout and the launch-page status card. split — when truthy, lay the metadata sections out in two equal columns that collapse to one column under the ``md`` breakpoint. Only meaningful in the full-width stacked layout; the narrow sidebar card leaves it falsy. {% endcomment %} {% if not bare %}
