{% load i18n %} {% comment %} Stacked, full-width report layout (the default). Three full-width (``col-12``) sections, in order: Run Summary, User Inputs, Outputs. - Run Summary is a plain card (header + two-column body + footer). Its metadata is laid out in two equal columns that stack on mobile (``split=True``), and the "View report as JSON" action lives in the card footer. - User Inputs and Outputs are each their own independent Bootstrap accordion so collapsing one never affects the other (no shared ``data-bs-parent``). Outputs defaults open; User Inputs defaults collapsed. They are included with ``bare=True`` so they emit only their inner content — the accordion supplies the chrome, avoiding a duplicated section title. Selected by ``validation_detail.html`` by default. The original two-column design lives in ``layout_validation_run_classic.html`` for a future toggle. Context variables: run, step_runs, step_outputs, step_params, step_template_warnings, artifact_items, issued_credential, credential_download_url, evidence_artifact (all forwarded from the run detail view). {% endcomment %}
